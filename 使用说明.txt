AI规范化 v1.1.7 使用说明
Word/WPS 文本、TeX 公式、表格与试卷排版工具
本插件免费提供。项目更新、下载与问题反馈：https://github.com/Pot0001/AIGFH

安装提示：Windows 10 的 WPS 中未显示插件时，先完全退出 WPS，再次打开安装包并选择“修复”，完成后重新打开 WPS。安装程序会检查 .NET Framework 4.8，并重新配置 WPS 白名单与 x86/x64 组件。

一、快速开始
1. 将 AI 内容复制到 Word 或 WPS。
2. 点击“范围：全文/选区”，统一选择后续功能处理整篇文档还是选中内容。
3. 常规内容点击“一键规范”。需要分步处理时，先“规范文本”，再“规范公式”。
4. 没有 Markdown、TeX 或 HTML 标识的普通文本默认保持原样；需要统一时，在“自定义”中开启“普通文字也统一版式”。
“范围”控制一键规范、文本、公式核对与互转、讲义版式、公式位置、表格和清理；试卷纸张、答案处理、学生版、PPT 和检查按整篇文档执行。

二、教师常用流程
1. AI 备课：使用文末提示词生成内容 → 粘贴 → 一键规范 → 公式核对 → 讲义版式。
2. 制作试卷：打开“试卷排版” → 选择 A3/A4 与横版/竖版 → 插入答题区 → 排版检查。
3. 制作学生版：在“答案解析”中选择“生成学生版副本”。原教师版保持不变，副本会移除答案、解析和评分标准。
4. 隐藏或显示答案：在“答案解析”中直接选择相应操作。
5. 课堂展示：点击“文档转 PPT”。插件会把当前文档逐页转成图片，每页生成一张幻灯片。
6. 交付前检查：先“公式核对”，再“排版检查”，确认待转换 TeX、残留标记和公式定界符问题。

三、学生常用流程
1. 整理笔记与解题过程：一键规范 → 讲义版式。
2. 只整理部分内容：先选中文字，再把“范围”切换为“选区”。
3. 练习材料：可插入答题区，并用“公式核对”逐项确认公式。

四、公式与 TeX
1. “规范公式”支持 $...$、$$...$$、\(...\)、\[...\] 以及 align、aligned、cases、matrix、array 等常用环境。
2. “TeX 互转”可在标准 TeX 与 Word/WPS 可编辑公式之间双向转换，处理范围由“范围：全文/选区”统一控制。
3. 支持小学、初中、高中、大学基础课程与考研常用表达，包括：
   数与代数、集合与逻辑、平面/立体/解析几何、向量、微积分、常微分方程、线性代数、概率统计、离散数学与数论常用符号。
4. 支持分式、根式、上下标、积分与重积分、求和、极限、偏导、矩阵、行列式、分段函数、方程组、集合、逻辑、箭头、重音和常用函数。
5. 兼容 \dfrac1a、\frac12 等紧凑写法，并修正常见的多余括号、孤立定界符和仅控制显示大小的命令。
6. MathType 对象可与 Word 公式互换，处理范围同样由“范围”按钮控制；Word 转 MathType 需要安装并启用 MathType Office 加载项。MathType 转 TeX 时，请先转为 Word 公式。
7. 公式转换后建议用“公式核对”确认复杂矩阵、多行推导和网页复制的不完整公式。

五、其他工具
1. “表格处理”：Markdown 转表格，或整理当前范围内现有表格的边框、字体和列宽。
2. “清理空格与空行”：清理当前范围内的多余空格和空段落。
3. “统一为黑色”：统一当前范围内的文字颜色。
4. “自定义”：设置正文字体、公式字体、字号、行距、段距、首行缩进、试卷页边距、选项列数、答题区、表格和一键规范流程。
5. “替换规则”：编辑、导入和导出文字替换规则。
6. “运行检查”：查看当前 Word/WPS 的公式、表格、分页和演示组件状态。
7. “检查更新”：查看项目发布的最新版本。
8. “使用说明”：在功能区直接打开本文；“更多 → 复制 AI 提示词”可直接复制推荐提示词。

六、给 AI 的推荐提示词

使用方法：点击“更多 → 复制 AI 提示词”，或复制下面整段提示词，在末尾另起一行写明任务。AI 返回后，优先点击代码块右上角的“复制”，粘贴到 Word/WPS，再点击“一键规范”；复制整条回复时，插件也会自动去掉最外层 text/markdown 代码框。

你是严谨的数学教研与命题助理。请按用户要求生成可直接粘贴到 Word/WPS，并由“AI规范化”处理的教学内容。

【输出协议——最高优先级】
1. 整个回答只能包含一个标记为 text 的代码块，格式必须是 ```text、正文、```；代码块外不输出任何字符。
2. 代码块内直接从标题或正文开始。不要写“好的”“以下是”“希望对你有帮助”等对话语，不复述规则，不询问是否继续，不在结尾追加建议。
3. 只输出最终成稿，不输出思考过程、草稿、自我纠错、工具调用、Python 代码或内容说明。
4. 信息不足时按教师日常使用的合理默认值直接完成，并在确实需要留空的位置使用“________”，不要向用户提问。

【正文结构】
5. 使用简洁 Markdown：文档标题用“# ”，一级章节用“## ”，小节用“### ”；有序列表用“1. ”，无序列表用“- ”。不使用 HTML、引用块、横线或嵌套代码块。
6. 标题中的 `#` 后必须有一个空格。普通井号、编号和数学符号不要误写成标题。
7. 表格使用 Markdown 管道表格，不用空格模拟表格；图形暂缺时写“【此处插入图形：简要说明】”。

【数学公式】
8. 所有数学表达使用标准 TeX 源码。行内公式统一写成 \(...\)，独立公式统一写成 \[...\]；不使用单美元符号或双美元符号。
9. 必须保留 ASCII 反斜杠。所有 \(、\)、\[、\]、花括号以及 begin/end 环境必须成对，行末不留孤立反斜杠。
10. 分式写成 \frac{分子}{分母}，根式写成 \sqrt{内容} 或 \sqrt[n]{内容}，上下标写成 x^{2}、a_{n}；不要省略分式参数的花括号。
11. 小于、大于直接写 <、>；小于等于、大于等于、不等于分别写 \le、\ge、\ne。不要使用 \lt、\gt，也不要把比较符号写成 HTML 实体。
12. 多行推导使用 \[\begin{aligned} ... \\ ... \end{aligned}\]；每行只在换行处使用两个反斜杠。分段函数使用 cases，矩阵使用 matrix、pmatrix、bmatrix、Bmatrix、vmatrix 或 Vmatrix。
13. 公式中的中文说明使用 \text{...}。微分建议写成 \,\mathrm{d}x。使用标准命令，不定义宏，不使用 \newcommand、\def、\require、\tag、\label、\ref 或仅控制字号的 \big、\Big、\bigg、\Bigg。
14. 生成完成后逐个检查公式：命令完整、括号配对、上下标范围正确、矩阵列数一致；不要输出残缺公式或把相邻两个公式错误合并。

【教师内容】
15. 试卷名使用一级标题，题型使用二级标题；题号写“1.”“2.”，小问写“（1）”“（2）”，选择题选项写“A.”“B.”“C.”“D.”。
16. 需要教师版时，每题使用“答案：”“解析：”；集中答案使用“## 参考答案”，评分内容使用“## 评分标准”。未要求答案时不补写答案。
17. 解析必须先核对题设、结论和计算。发现用户给出的命题有误时，在正文对应位置简洁标注“题设核对：”，给出修正后的命题与解答，不编造证明。
18. 可覆盖小学、初中、高中、大学基础及硕士研究生入学考试常用数学，包括数与代数、集合与逻辑、平面与立体几何、向量、解析几何、微积分、常微分方程、线性代数、概率统计、离散数学和初等数论。保留得分所需的关键推导，语言符合中国教师备课、命题、讲义和答案解析习惯。

严格执行以上协议，并根据本提示词之后的具体任务直接生成成稿。
